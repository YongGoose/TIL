# CHAPTER 05. 컨텍스트를 가져오는 기술 - 벡터 서치

## LLM의 숨은 영웅, 임베딩

- **임베딩(Embedding)**: 텍스트, 이미지 등 비정형 데이터를 고차원 벡터(숫자 배열)로 변환하는 기술
  - 단어, 문장, 문서의 **의미를 수치화**하여 컴퓨터가 이해할 수 있는 형태로 표현
  - 의미가 유사한 텍스트는 벡터 공간에서 가까운 위치에 배치됨
- 임베딩이 중요한 이유
  - LLM은 학습 데이터에 포함되지 않은 정보(최신 정보, 사내 문서 등)에 대해 정확한 답변이 어려움
  - 임베딩을 활용하면 **외부 지식을 LLM에 효과적으로 전달**할 수 있음
  - RAG(Retrieval-Augmented Generation) 등 핵심 기술의 기반이 됨
- 임베딩 모델의 작동 방식
  - 텍스트를 입력하면 수백~수천 차원의 벡터를 출력
  - 예: "강아지"와 "개"는 벡터 공간에서 가까이 위치, "강아지"와 "자동차"는 멀리 위치
  - OpenAI, Google, Cohere 등에서 다양한 임베딩 모델을 제공

## 벡터 서치와 시맨틱 서치

- **키워드 서치의 한계**
  - 전통적인 검색은 정확한 단어 일치에 의존
  - "LLM 비용 절감 방법"을 검색했을 때, "토큰 사용량을 줄이는 전략"이라는 문서는 키워드가 달라 검색되지 않음
- **벡터 서치(Vector Search)**: 임베딩 벡터 간의 유사도를 계산하여 의미적으로 가장 관련 있는 데이터를 찾는 검색 방식
  - 코사인 유사도(Cosine Similarity) 등을 사용하여 벡터 간 거리를 측정
  - 단어가 정확히 일치하지 않아도 **의미가 유사하면 검색 결과에 포함**
- **시맨틱 서치(Semantic Search)**: 벡터 서치를 활용한 의미 기반 검색
  - 사용자의 질문 의도를 이해하고, 의미적으로 관련된 문서를 검색
  - 기존 키워드 검색 대비 검색 품질이 크게 향상됨
- **벡터 데이터베이스**: 벡터 서치를 효율적으로 수행하기 위한 전용 데이터베이스
  - Pinecone, Weaviate, Chroma, Milvus 등이 대표적
  - 대규모 벡터 데이터에서 빠른 유사도 검색을 지원

## 벡터 서치의 명암

- **장점**
  - 의미 기반 검색으로 키워드 불일치 문제 해결
  - 다국어 검색 지원: 언어가 달라도 의미가 같으면 유사한 벡터로 매핑
  - RAG 파이프라인의 핵심 구성 요소로 LLM의 환각(Hallucination) 감소에 기여
- **한계와 주의점**
  - 임베딩 모델의 품질에 따라 검색 정확도가 크게 좌우됨
  - 짧은 텍스트 vs 긴 텍스트 간의 임베딩 품질 차이가 존재
  - **청킹(Chunking)** 전략이 중요: 문서를 어떤 단위로 나누느냐에 따라 검색 성능이 달라짐
  - 벡터 서치만으로는 부족한 경우가 있어, 키워드 검색과 결합한 **하이브리드 서치**가 실무에서 많이 사용됨
- 벡터 서치는 도구일 뿐, **프롬프트에 적절한 컨텍스트를 제공하는 것**이 최종 목표
  - 좋은 컨텍스트 = LLM이 더 정확하고 유용한 답변을 생성하는 기반
