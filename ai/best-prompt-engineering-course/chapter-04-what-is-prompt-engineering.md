# CHAPTER 04. 프롬프트 엔지니어링이란

## AI 시대 새로운 코딩, 프롬프트 엔지니어링

- **프롬프트(Prompt)**: LLM에게 전달하는 입력 텍스트(질문, 지시, 맥락 등)
- **프롬프트 엔지니어링**: LLM에서 원하는 결과를 얻기 위해 프롬프트를 체계적으로 설계하고 최적화하는 기술
- 프로그래밍 언어 대신 **자연어가 새로운 인터페이스**가 됨
  - 같은 의도라도 프롬프트를 어떻게 작성하느냐에 따라 결과가 크게 달라짐
- 프롬프트 엔지니어링이 중요한 이유
  - LLM의 잠재력을 최대한 끌어내기 위해 필수
  - 비용 절감: 효율적인 프롬프트로 API 호출 횟수와 토큰 사용량 감소
  - 품질 향상: 환각(hallucination) 감소, 정확도 및 일관성 향상

## 프롬프트 엔지니어링의 가장 대표적인 다섯 가지 방법

- **Zero-shot Prompting**: 예시 없이 지시만으로 태스크 수행
  - 예: "다음 문장의 감정을 분석해줘"
- **Few-shot Prompting**: 소수의 예시를 제공하여 원하는 출력 형식과 패턴을 유도
  - 예: 입력-출력 쌍을 2~3개 보여준 뒤 새로운 입력에 대한 출력을 요청
- **Chain of Thought (CoT)**: "단계별로 생각해봐"와 같이 추론 과정을 유도
  - 복잡한 수학, 논리 문제에서 정확도가 크게 향상됨
- **Role Prompting**: LLM에게 특정 역할을 부여하여 답변의 관점과 톤을 조절
  - 예: "너는 10년 경력의 시니어 개발자야"
- **Instruction Prompting**: 명확하고 구체적인 지시사항을 제공
  - 출력 형식, 길이, 제약 조건 등을 명시적으로 지정

## 기본적인 프롬프트 구성

- 좋은 프롬프트의 핵심 요소
  - **역할(Role)**: LLM이 어떤 관점에서 답변할지 설정
  - **맥락(Context)**: 배경 정보나 상황 설명 제공
  - **지시(Instruction)**: 수행해야 할 태스크를 명확하게 기술
  - **입력 데이터(Input)**: 처리할 대상 데이터
  - **출력 형식(Output Format)**: 원하는 답변의 형태를 지정 (JSON, 표, 리스트 등)
- 프롬프트 작성 팁
  - 모호한 표현보다 **구체적이고 명확한 지시**를 사용
  - 한 번에 너무 많은 것을 요구하지 않고 **태스크를 분리**
  - 원하지 않는 것을 말하기보다 **원하는 것을 직접 명시**

## 프롬프트 엔지니어링 원 포인트 레슨

- **반복 실험(Iteration)이 핵심**: 한 번에 완벽한 프롬프트를 만들기 어려움
  - 결과를 확인하고 프롬프트를 조금씩 수정해가며 최적화
- **구분자(Delimiter) 활용**: 입력 데이터와 지시를 명확히 분리
  - 예: ```, ---, """ 등으로 섹션 구분
- **출력 길이 제어**: "3문장으로 요약해줘", "100자 이내로 작성해줘" 등 명시
- 모델의 한계를 이해하고 활용하기
  - 최신 정보가 필요하면 RAG 등 외부 지식 연동 활용
  - 복잡한 계산은 코드 실행 도구와 결합하여 정확도 향상
- **프롬프트 엔지니어링은 기술이자 예술**: 체계적 방법론 + 창의적 실험이 모두 필요
